# geo-rewrite 사용자 가이드 (Desktop 버전)

**버전:** v1.0
**대상:** Claude Desktop / claude.ai 사용자
**목적:** 원본 콘텐츠를 SEO/GEO 최적화 기준으로 분석·재작성(윤문)

---

## 1. 이 스킬은 무엇인가

`geo-rewrite`는 웹페이지 URL, 콘텐츠 파일, 또는 붙여넣은 본문을 입력받아 다음을 자동으로 합니다.

- **분석**: 목표별 현황 진단 + 개선 전략·우선순위 제안
- **재작성**: 윤문 체크리스트를 적용한 최적화 원고 + 변경 요약 생성

Tunnel Framework 콘텐츠 파이프라인의 최적화 규칙을 **콘텐츠 한 편에 바로 적용하는 Minimal 버전**입니다. 스크립트 없이 순수 프롬프트로 동작합니다.

---

## 2. 대상 사용자

- **마케터**: 기존 글을 AI 검색·SEO에 맞게 리프레시할 때
- **라이터**: 초안이나 클라이언트 피드백을 반영해 원고를 다듬을 때
- **콘텐츠 팀**: 발행 전 품질·신뢰·전환 동선을 점검할 때

---

## 3. 설치

1. `geo-rewrite-desktop.zip` 파일을 준비합니다.
2. **설정 → Capabilities(기능) → Skills** 로 이동합니다.
3. **스킬 업로드(Upload skill)** 를 눌러 `geo-rewrite-desktop.zip` 을 선택합니다.
4. 업로드가 완료되면 대화에서 바로 사용할 수 있습니다.

> Skills는 유료 플랜(Pro/Max/Team/Enterprise)에서 제공됩니다. 메뉴 명칭은 앱 버전에 따라 다를 수 있습니다.

---

## 4. 입력 3가지 방식

Desktop에서는 로컬 파일 경로에 접근할 수 없으므로 아래 중 하나로 입력합니다.

- **URL**: `https://example.com/post 윤문해줘`
- **파일 첨부**: 원고(.md/.html/.txt)를 첨부하고 "SEO 최적화로 다듬어줘"
- **본문 붙여넣기**: 글 본문을 붙여넣고 "이거 분석부터 해줄 수 있어?"

---

## 5. 두 가지 모드

### 분석 모드 — 어떻게 고칠지 먼저 본다

```
이 글 SEO 중심으로 분석해줄 수 있어?
GEO랑 전환 우선으로 진단해줘
```

결과 예:
```
# 최적화 분석

## SEO (65/100)
- 제목: 키워드 부재 → "AI 코딩" 추가 권장
- 메타: description 없음 → 신규 작성
- FAQ: 없음 → 2개 추가 가능

## GEO (50/100)
- 출처 링크 3/5 미보유 → 1차 링크 보강
- 직접 인용 부족(패러프레이징 과다)

## 종합 우선순위
1. 제목 키워드 + 110자 정리
2. FAQ 추가(AI 인용 신호)
3. 출처 링크 보강
```

### 재작성 모드 — 바로 윤문본을 만든다

```
바로 재작성해줘. SEO·GEO·전환 목표로.
분량 20% 줄여서 윤문본 만들어줘.
```

결과: 윤문 본문 + 변경 요약 + 핵심 메타(제목·description·FAQ·출처). "파일로 줘"라고 하면 `.md`로 다운로드.

---

## 6. 목표별 초점과 개선 예시

### SEO
제목(≤110자·키워드)·메타 설명·FAQ·H2/H3 계층.
```
"AI 코딩 도구란?"  →  "Claude Code AI 코딩으로 개발 속도 높이기 — 입문 가이드"
```

### GEO (AI 검색 인용)
출처 명시·passage 자기완결·엔티티 정의.
```
"Claude는 AI 모델입니다."(출처 없음)
→ "Claude는 Anthropic이 개발한 대규모 언어 모델(LLM)입니다 [Anthropic 공식 문서]."
```

### 톤
과장 제거·조건부 표현·중립성.
```
"가장 빠른 도구입니다!"  →  "개발 생산성을 높일 수 있는 도구입니다(결과는 환경에 따라 다를 수 있습니다)."
```

### 명확성
Answer-First·짧은 문장·용어 일관성. 긴 나열 문장을 핵심 먼저 + 리스트로 정리.

### 분량
중복 병합·예시 정리(압축) 또는 근거·사례 보강(확장).

### AI 탐지 회피
기계적 나열·저자 부재 → 1차 관점·구체 사례.
```
"여러 이점이 있습니다. 첫째… 둘째… 셋째…"
→ "6개월 써보니 가장 큰 변화는 반복 코드 작성 시간이 줄어든 점입니다. 예를 들어…"
```

### 전환
본문 논리가 자연스럽게 CTA로 이어지는 브릿지 + 행동 유도.

### 고유성
원본의 1차 경험·차별 각도를 살려 "이 글이 없어도 똑같은 답이 나오는" 일반론을 피함.

## 7. 권장 흐름

```
1. "분석해줄 수 있어?"  → 우선순위 확인
2. "좋아, 이대로 재작성해"  → 윤문본 생성
3. "파일로 줘"  → 다운로드·저장
4. 검수 후 발행
```

## 8. 자주 묻는 질문

**Q. 분석과 재작성 중 뭘 먼저?**
분석 → 재작성 순서를 권합니다. 우선순위를 보고 꼭 필요한 부분만 재작성할 수 있습니다.

**Q. 없는 내용을 채워주나요?**
아니요. 본문에 없는 사실·수치·사례는 지어내지 않고 `TODO: 확인 필요`로 표시합니다. 직접 채워 넣으세요.

**Q. 원본 내용이 바뀌나요?**
핵심 주장·사실·고유 경험은 보존하고 표현·구조·신호만 최적화합니다. 변경한 내용은 변경 요약으로 알려줍니다.

**Q. 페르소나·CEP 기반 최적화도 되나요?**
이 스킬은 글 한 편에 내재한 규칙만 적용하는 Minimal 버전입니다. 프로젝트 데이터(페르소나·CEP·코퍼스)가 필요한 최적화는 Tunnel Framework `/cp` 파이프라인에서 처리합니다.

**Q. 로컬 파일 경로를 넣으면 되나요?**
Desktop은 서버 샌드박스라 PC 파일 경로에 접근할 수 없습니다. 첨부하거나 본문을 붙여넣어 주세요.

---

## 9. 관련 도구 (Tunnel Framework)

| 도구 | 역할 |
|------|------|
| `/cp` | CP 파이프라인(콘텐츠 생성·검증·배포, 풀 버전) |
| `/schema-gen` | 구조화 데이터(JSON-LD) 생성 |
| `/geo` | GEO 감사(최적화 효과 검증) |
