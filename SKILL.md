# geo-rewrite 스킬 (Desktop 버전)

원본 콘텐츠를 분석하고 SEO/GEO 최적화 기준으로 재작성(윤문)하는 순수 프롬프트 스킬.
**스크립트 없이 이 문서의 지침에 따라 직접 분석·재작성·검증한다.**

## 사용법

대화에서 아래 중 한 가지 방식으로 입력한다.

- **URL**: `https://example.com/post 윤문해줘`
- **파일 첨부**: 원고 파일(.md/.html/.txt)을 첨부하고 "SEO 최적화로 다듬어줘"
- **본문 붙여넣기**: 글 본문을 붙여넣고 "이거 분석부터 해줄 수 있어?"

옵션(자연어로 함께 전달):
- **모드**: "일단 분석만" → 진단·전략 제안 / "바로 재작성해" → 윤문본 생성
- **목표**: "GEO랑 전환 중심으로" → 해당 목표 우선 (미지정 시 SEO·GEO·톤·명확성 기본)
- **분량**: "20% 줄여서" / "더 자세히"

> Desktop 환경에서는 사용자 PC의 로컬 파일 경로(`/Users/...`)에 접근할 수 없다. 파일은 **첨부**하거나 본문을 **붙여넣어** 전달하고, 윤문본은 다운로드받아 저장한다.

## 지원 목표 (8종)

| 목표 | 초점 |
|------|------|
| SEO | 제목(≤110자·키워드)·메타 설명·FAQ·H2/H3 계층 |
| GEO | AI 검색 인용 — 출처 명시·passage 자기완결·엔티티 정의 |
| 톤 | 신뢰감·전문성·중립성(과장 제거·조건부 표현) |
| 명확성 | Answer-First·문장 길이·용어 일관성 |
| 분량 | 압축·확장(중복 병합·근거 보강) |
| AI 탐지 | 1차 관점·구체 사례·자연발화(기계적 패턴 제거) |
| 전환 | 본문→CTA 브릿지·행동 유도·관련 링크 |
| 고유성 | 1차 경험·차별 각도(originality) |

---

## 실행 절차

### 1단계 — 콘텐츠 로드

- **URL**: 웹에서 본문·제목(H1)·소제목 구조·기존 메타(title·description·저자·발행일)를 추출한다. 본문이 비고 메뉴만 오면 "JS 렌더링이라 추출 제한 — 첨부/붙여넣기 권장"을 안내하고 추출 범위 내에서만 진행한다.
- **첨부/붙여넣기**: 전달된 내용을 직접 분석한다. 마크다운 frontmatter가 있으면 메타데이터로 참고한다.

### 2단계 — 모드 판정

- "분석"·"진단"·"어떻게 고칠지" → **분석 모드**
- "재작성"·"윤문"·"다시 써" → **재작성 모드**
- 불명확하면 **분석 → 재작성** 순서를 권하고 분석부터 한다.

### 3단계A — 분석 모드

선택된(또는 기본) 목표별로 현황을 진단하고 개선 방안·우선순위를 제시한다.

```
# 최적화 분석 — {제목}

## SEO   (현황 점수 N/100)
- 강점: ...
- 개선점: ...  → 실행 방안: ...

## GEO   (N/100)
...

## 종합 우선순위
1. {최우선} — {이유}
2. ...
```

### 3단계B — 재작성 모드

**이 모드의 산출물은 완성된 윤문 원고 전문(全文)이다.** 아래 체크리스트는 *적용 기준*일 뿐, 출력은 점검 목록이나 발췌가 아니라 **처음부터 끝까지 다시 쓴 본문 전체**여야 한다. 도입부터 FAQ·출처까지 한 편을 완결해서 그대로 출력한다. "이런 점을 고치면 됩니다" 식 안내만 내놓지 않는다.

아래 **윤문 체크리스트**를 적용해 본문을 재작성한다.

**구조**
- 본문에 H1(`# 제목`) 쓰지 않는다(발행 시스템이 제목을 H1로 렌더링 → 이중 H1 방지)
- 모든 H2는 질문형(`## ~인가요?`, `## ~하는 방법은?`)
- 각 H2 첫 문단에서 그 질문에 직접 답한다(Answer-First, 그 문단만 읽어도 답이 되는 자기완결)
- 글 끝에 FAQ 5~10개(질문형 + 2~3문단 답변). 단, **진짜 FAQ만** — 본문에 없는 질문을 지어내지 않는다
- 불릿 남발 금지: 비교·대조는 표, 흐름은 산문
- 줄바꿈 없는 숫자 나열 금지
- 각주 정의(`[^n]:`)는 본문 사이가 아니라 글 하단 "출처" 섹션에 모은다

**GEO/AIO**
- 모든 수치·주장에 출처 명시(공신력 있는 1·2차 출처 우선). 가상 URL 금지
- 인물·기관·개념·제품은 처음 등장 시 정의, 필요하면 동의어 병기("드랍쉬핑(위탁판매)")
- 연도를 명시하되 미래 연도를 과거처럼 쓰지 않는다

**SEO**
- 제목 110자 이내 + 핵심 키워드 포함
- 메타 설명(description) 150~160자
- H2/H3 계층 최대 3단

**품질·신뢰(E-E-A-T·규제·originality)**
- 저자·출처 신뢰 신호 보강(있으면 유지, 없으면 보강 제안)
- 업종별 규제 표현 회피 + 필수 고지(아래 표)
- 자사/미검증 수치는 `※ 측정 평균 — 보장 아님` 류 면책 부기
- 고유성: 원본의 1차 경험·사례·차별 각도를 살린다. "이 글이 없어도 LLM이 똑같은 답을 낼까?"가 "아니오"가 되도록

**톤(자연발화·AI 탐지 회피)**
- 메타 서술 금지("이 글에서는 ~ 알아봅니다")
- 모호한 추측으로 문장을 끝맺지 않는다(판단·기준·다음 단계로 마무리)
- 종결어미·문장 길이를 변주해 리듬을 준다
- 단락 길이 상한(도입 ~150자 / H2 첫 단락 ~200자 / 본문 ~240자)
- 1차 관점·구체 사례로 추상적 나열을 대체

**업종별 규제 표현 (해당 시)**

| 업종 | 회피 | 필수 고지 |
|------|------|----------|
| 의료 | "치료·예방·효과적" 단정 | "전문의 상담 권장" |
| 금융 | "원금 보장·무조건 수익" | "손실 위험·결과 보장 안 함" |
| 교육 | "100% 합격·무조건 취업" | "결과는 개인에 따라 다름" |
| 법률·세무 | "반드시·무조건 승소" | "전문가 자문 권장" |

### 4단계 — 출력 (전문 우선)

아래 **순서**로 출력한다. ①이 핵심 산출물이며 절대 생략·발췌하지 않는다.

**① 윤문 원고 전문** — 재작성된 본문 전체를 마크다운으로 처음부터 끝까지 출력한다(도입~FAQ~출처). 이 부분이 비거나 일부만 있으면 실패다.

```
(여기에 완성된 윤문 본문 전체를 그대로)
```

**② 변경 요약** — 무엇을 왜 바꿨는지.
```
## 변경 요약
- 글자수: N → N (±N%)
- SEO: 제목 키워드 추가 · 메타 신규 · FAQ N개
- GEO: 출처 N개 보강 · 엔티티 정의 추가
- 톤/AI탐지: 메타 서술 제거 · 1차 관점 추가
```

**③ 핵심 메타**
```
## 핵심 메타
- 제목 / description(150~160자) / FAQ N개 / 출처 목록
```

"파일로"라고 하면 ①(윤문 전문)을 다운로드 가능한 `.md`로 제공한다.

## 핵심 원칙

- **추정 금지**: 본문에 없는 사실·수치·사례·인용을 지어내지 않는다. 보강이 필요하면 `TODO: 확인 필요`로 표시하고 사용자에게 알린다.
- **원본 보존**: 원본의 핵심 주장·사실·고유 경험을 바꾸지 않는다. 표현·구조·신호만 최적화한다.
- **변경 이유 명시**: 무엇을 왜 바꿨는지 변경 요약에 적는다.
- **규제 점검**: 업종이 파악되면 금지 표현·필수 고지를 점검한다.

## 주의 사항

- **Minimal 범위**: 이 스킬은 콘텐츠 한 편에 내재한 규칙만 적용한다. 페르소나 tier 톤·CEP 차원·프로젝트 코퍼스 대비 near-duplicate 등 **프로젝트 데이터가 필요한 최적화는 범위 밖**이며 Tunnel Framework `/cp` 파이프라인에서 처리한다. (글 내부 H2 중복은 점검한다.)
- **리치 결과 ≠ AI 인용**: 구조화·FAQ가 검색 리치 결과로 보장되지 않지만, AI 검색(ChatGPT·Perplexity·Google AI 개요 등)의 이해·인용에는 유효하다 — 그것이 GEO 목적이다.
- **표기**: 한국어 콘텐츠는 한글로 작성(불필요한 한자 혼입 금지), 이모지 미사용.

## 성공 기준 / 오류 처리

| 상황 | 성공 기준 |
|------|----------|
| 분석 모드 | 목표별 진단 + 종합 우선순위 출력 |
| 재작성 모드 | **윤문 원고 전문(전체 본문)** 우선 출력 + 변경 요약 + 핵심 메타. 점검 목록·발췌만 내면 실패 |
| 파일 요청 | 위 + 다운로드 가능한 .md |

| 오류 | 대응 |
|------|------|
| URL 본문 추출 실패 | 경고 후 첨부/붙여넣기 권장, 추출 범위 내 진행 |
| 보강 근거 없음 | 날조 대신 `TODO: 확인 필요` 표시 |
| 모드 불명확 | 분석 → 재작성 순서 권유 |
